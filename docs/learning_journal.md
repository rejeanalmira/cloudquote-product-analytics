# Sprint 2 - Product Health Analysis

## Objective

Analyze CloudQuote support ticket data to identify trends in product health and recommend areas for product improvement.

## What I Completed

- Imported support ticket data into MySQL.
- Fixed a schema mismatch by changing `ticket_id` from `INT` to `VARCHAR`.
- Created SQL queries using:
  - COUNT()
  - AVG()
  - GROUP BY
  - ORDER BY
- Identified ticket trends by:
  - Issue Type
  - Product Area
  - Customer Segment
  - Severity
  - Region
- Wrote executive findings document summarizing business insights and recommendations.

## Technical Skills Practiced

- MySQL Workbench
- SQL Aggregations
- Git & GitHub
- VS Code
- Business Analysis
- Data Storytelling

## Lessons Learned

- Always verify that table schemas match the source data before importing.
- Use descriptive aliases to improve query readability.
- SQL answers business questions. Analysts explain why the results matter
